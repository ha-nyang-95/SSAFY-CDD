# 프로젝트 요약

## 1. 프로젝트 개요
- **프로젝트명**: ODD (CRACK DETECTION DRONE)
- **핵심 내용**: AI 드론을 기반으로 시설물의 균열을 실시간으로 탐지하고, 그 결과를 3D 모델로 시각화하여 제공하는 시스템입니다. 기존의 육안 검사 방식은 많은 시간과 비용, 인력이 소요되고 작업자 안전사고의 위험이 높으며, 검사 주기가 길어 조기 발견이 어려운 문제점이 있었습니다. 이 프로젝트는 AI와 드론 기술을 활용하여 이러한 문제들을 해결하고, 안전 점검의 새로운 패러다임을 제시하는 것을 목표로 합니다.

## 2. 주요 목표
- **실시간 균열 탐지 (Crack Detection)**: 드론이 촬영하는 영상을 On-Device AI(YOLO)를 통해 실시간으로 분석하여 균열을 탐지하고 바운딩 박스로 표시합니다.
- **선택적 정밀 검사 (Crack Segmentation)**: 사용자가 필요하다고 판단한 균열에 대해 AI(Segmentation)와 LiDAR 센서를 이용하여 깊이, 폭 등 상세 데이터를 정밀하게 측정합니다.
- **3D 모델링 및 시각화 (3D Modeling)**: 수집된 2D 이미지들을 종합하여 구조물의 3D 모델을 생성하고, 탐지된 균열의 위치, 심각도, 정밀 측정 데이터를 3D 모델 위에 매핑하여 직관적인 리포트를 제공합니다.

## 3. 기술 스택 및 방법론
- **AI 모델**:
    - **YOLOv8n**: 실시간 균열 탐지를 위해 사용되는 On-Device AI 모델입니다.
    - **Image Segmentation**: 균열의 형태를 정밀하게 분석하고 폭, 크기 등의 데이터를 측정하는 데 사용됩니다.
- **하드웨어**:
    - **드론**: 영상 촬영 및 센서 데이터 수집을 위한 기본 플랫폼입니다.
    - **LiDAR 센서**: 균열의 깊이 등 정밀 데이터 측정을 위해 활용됩니다.
    - **NVIDIA Jetson**: On-Device AI 모델을 탑재하고 실시간 추론을 수행하는 임베디드 보드입니다.
- **개발 및 방법론**:
    - **AIoT**: AI와 사물인터넷(드론)을 결합한 접근 방식을 채택합니다.
    - **웹 서비스**: 분석된 데이터를 사용자가 확인할 수 있는 웹 기반 대시보드를 풀스택으로 개발합니다.
    - **데이터 시각화**: 탐지된 균열 정보를 3D 모델에 매핑하여 사용자의 직관적인 이해를 돕습니다.

## 4. 예상 결과 및 영향
- **비용 및 시간 절감**: 자동화된 탐사 및 분석을 통해 기존 육안 검사 방식 대비 90% 이상의 효율 증대를 기대합니다.
- **작업자 안전 확보**: 교량, 댐 등 위험 지역에 작업자가 직접 접근할 필요 없는 비대면 점검으로 중대재해 발생 가능성을 원천적으로 차단합니다.
- **데이터 기반 유지보수**: 균열 데이터를 3D 모델과 함께 영구적으로 보존하고 변화 추이를 추적하여 정량적이고 체계적인 유지보수 계획 수립이 가능해집니다.
- **업무 효율성 증대**: 사용자가 중요 결함에만 집중하여 선택적으로 정밀 검사를 수행함으로써 전체 점검 프로세스를 최적화할 수 있습니다.

## 5. 시스템 아키텍처
- **데이터 수집 (드론 및 엣지 디바이스)**:
    - 드론의 카메라와 LiDAR 센서는 영상 및 센서 데이터를 수집하여 라즈베리파이로 전송합니다.
    - 라즈베리파이는 컨트롤 모듈로서, 수집된 영상은 RTSP로, LiDAR 제어 신호는 MQTT 프로토콜을 통해 Jetson Orin Nano에 전달합니다.

- **실시간 처리 및 클라우드 전송 (Jetson Orin Nano)**:
    - Jetson Orin Nano는 엣지 디바이스의 핵심으로, 수신한 영상 스트림을 RTMP를 통해 AWS S3로, WebRTC를 통해 웹 프론트엔드(FE)로 실시간 전송합니다.
    - 현장 관리자는 유선으로 연결된 LCD를 통해 실시간 확인이 가능합니다.

- **AI 분석 및 데이터 처리 (클라우드)**:
    - AWS S3에 저장된 영상과 이미지는 AI 분석(Segmentation, 3D 모델링)을 위해 Runpod 서버리스 환경으로 전달되고, 처리된 결과는 다시 S3에 저장됩니다.
    - 데이터가 S3에 업로드되면 AWS Lambda 함수가 트리거되어 EC2에 배포된 백엔드(BE) 서버와 미디어 서버에 알림을 보냅니다.

- **웹 서비스 및 사용자 인터페이스 (EC2 및 관리자)**:
    - EC2 인스턴스에는 프론트엔드(React + Nginx), 백엔드(Spring Boot), 데이터베이스(DB), 미디어 서버가 구성됩니다.
    - 안전 관리자는 웹(shyo2.com)을 통해 상세 분석 결과를 확인하고 컨트롤러로 드론을 직접 조종합니다.

## 6. 사용자 플로우차트
- **촬영 준비 및 실시간 탐지**:
    - 사용자가 LCD 화면에서 '촬영 시작 버튼'을 누르면 라즈베리파이(Pi4)가 카메라 촬영을 시작하고 Jetson으로 영상을 전송합니다.
    - Jetson은 실시간 균열 탐지를 수행하고, 이 과정에서 생성된 원본 영상(Raw 영상)은 S3에 저장됩니다. 동시에 사용자는 웹(FE)을 통해 실시간 영상을 확인할 수 있습니다.
- **정밀 탐지**:
    - 사용자가 '정밀 탐지 버튼'을 누르면 라즈베리파이는 LiDAR를 동작시켜 Raw 이미지와 LiDAR 값을 수집합니다.
    - 이 데이터가 S3에 저장되면, 변경 사항을 감지한 AWS Lambda가 AI 모델을 트리거하여 'Segment 처리'를 수행하고 결과를 다시 S3에 저장합니다.
- **촬영 종료 및 3D 모델링**:
    - '촬영 종료 버튼'을 누르면 최종 영상이 S3에 저장됩니다.
    - Lambda가 이 변화를 감지하여 AI 모델을 통해 '3D AI 처리'를 실행하고, 생성된 3D 모델을 S3에 저장합니다.
- **결과 확인**:
    - 3D 모델 저장이 완료되면 Lambda는 웹 백엔드(BE)로 작업 정보를 전송합니다.
    - 백엔드는 이 정보를 데이터베이스(DB)에 저장한 후, 프론트엔드(FE)로 최종 데이터를 보냅니다.
    - 사용자는 웹 화면에서 최종 3D 모델과 정밀 탐지 결과를 확인할 수 있습니다.
